# Code of conduct

Applied AI Rig contributors are expected to keep project spaces constructive, technically honest, and safe
for participation.

- Discuss ideas and code rather than attacking people.
- Welcome questions and different experience levels without dismissive language.
- State uncertainty, evidence limits, conflicts of interest, and material tradeoffs.
- Do not publish another person's private information, credentials, project data, prompts, or model output.
- Do not use project channels for harassment, discrimination, threats, sexualized attention, or sustained
  disruption.

Maintainers may edit or remove contributions and restrict participation when behavior violates these
expectations. Reports should be sent privately to the repository owner using an available private GitHub
channel. Do not include sensitive evidence in a public issue.

Enforcement decisions should consider context, impact, repetition, and the safety of affected participants.
Good-faith technical disagreement and evidence-backed criticism are welcome.
