# Support

Use GitHub issues for reproducible defects and scoped feature requests after the repository is public. Use
the module-proposal form only when an applied-AI risk cannot be represented by the current core and five
modules.

Before reporting a problem:

1. Run `python3 init.py --check <target>` and include its safe output.
2. Include the Applied AI Rig version, Python version, operating system, command, and minimal reproduction.
3. Remove secrets, personal data, prompts, responses, private endpoints, account identifiers, and
   confidential project content.

Security vulnerabilities do not belong in public issues. Follow [SECURITY.md](SECURITY.md).

The project cannot provide legal advice, certify compliance, review private datasets, or operate a user's
application. Questions should remain focused on the installer, checker, generated engineering records, and
documented interoperability boundaries.
